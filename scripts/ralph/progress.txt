## 2026-03-18 вЂ” US-009: Deploy Cart Transform and verify end-to-end вњ…

**Root cause discovered:** Shopify Cart Transform (`purchase.cart-transform.run`) requires Shopify Plus. The store is on Basic plan вЂ” the function was silently no-oping, so checkout showed в‚є230 (base price) instead of the calculated price.

**Solution: Draft Orders API (available on all plans)**
- Pivoted from Cart Transform to server-side Draft Order creation via PMS backend
- New endpoint: `POST /api/checkout/draft-order` in `ecommerce/pms/src/routes/checkout.ts`
  - Validates: variantId, en (50вЂ“1000 cm), boy (1вЂ“600 cm), pileOrani (2.0/2.5/3.0), kanatCount (1 or 2)
  - Fetches short-lived Shopify token via `client_credentials` OAuth flow
  - Fetches variant base price from Shopify REST API (server-side вЂ” never trusts client price)
  - Calculates: `(en / 100) Г— pileOrani Г— kanatCount Г— basePricePerMeter`
  - Creates Shopify Draft Order with correct price, order properties (En, Boy, Pile Stili, Kanat)
  - Returns `{ checkoutUrl: draft_order.invoice_url }`
- PMS server.ts updated to register `checkoutRouter` at `/api/checkout`
- `env.ts` updated with `getShopifyClientCredentials()` helper
